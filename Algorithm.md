## 数据结构
```
程序的本质
    程序为了具体问题而存在
    程序围绕如何解决问题进行设计
    同一个问题有多种解决方案
    如何追求程序的"性价比"?
    是否有可量化的方法判别程序的好坏?
数据结构起源
    计算机从解决数值计算问题到解决生活中的问题
    现实生活中的问题涉及不同个体间复杂联系
    需要计算机程序中描述问题中个体间的联系
    数据结构主要研究（非数值计算程序问题中的操作对象之间的）关系
    不是研究复杂的算法
数据
    程序的操作对象，用于描述客观事物
    数据是一个抽象的概念，将其进行分类后得到程序语言中的类型，例如：int,double,其它自定义类型等等
数据元素
    组成数据的基本单位，简体理解就是集合里的一个元素，例如：lst[1]就是一个数据元素
    数据元素之间不是独立的，存在特定的关系，这些关系就是数据结构
    例如：数组中各个元素之间存在固定的线性关系
数据项
    一个数据元素由若干个数据项组成，例如：lst[1].age和lst[1].name就是2个数据项
数据对象
    性质相同的数据元素的集合
    关键点：是一个集合

|---------------------------------------------------------------------------------------------------|
|                                        数据                                                       |
|---------------------------------------------------------------------------------------------------|
|                数据对象                 |                数据对象                |  非数据对象      |
|----------------------------------------|----------------------------------------|                 |
|    数据元素        |      数据元素       |      数据元素      |         数据元素   |                 |
|—------------------|--------------------|--------------------|-------------------|                 |
|  数据项A | 数据项B |  数据项A | 数据项B  |  数据项A | 数据项B  |  数据项A | 数据项B |                 |
|---------------------------------------------------------------------------------------------------| 
数据结构
    研究数据元素之间的关系
    数据结构主要研究非数值计算问题中的操作对象以及它们之间的关系
    数据结构不是研究复杂的算法
数据的逻辑结构
    集合：没有关系，只是都属于同一个集合
    线性结构：1对1，如数组，线性表，栈，队列
    树形结构：1对N，如树
    图形结构：N对N，如图
数据的存储结构
    顺序存储：借助元素在存储器中的相对位置来表示元素数据之间的逻辑关系
    链式存储：借助指示元素存储地址的指针表示数据元素之间的逻辑关系
    索引：字典目录
    散列：
数据的逻辑结构与存储结构密切相关
    算法设计——>逻辑结构
    算法实现——>存储结构
算法
    算法是特定问题的求解步骤
    在计算机表现为指令的有限序列
    算法是独立存在的一种解决问题的方法和思想
    对于算法而言，语言并不重要，重要的是思想
算法和数据结构的区别
    数据结构只是静态的描述了数据元素之间的关系
    高效的程序需要在数据结构的基础上设计和选择算法
    算法是为了解决实际问题设计的
    数据结构是算法解决的载体
    数据结构与算法相辅相成
    程序=数据结构+算法
    面向对象=数据结构(侧重)+算法
    面向过程=算法(侧重)+数据结构
算法的特性
    具有0个或多个输入
    具有1个或多个输出
    算法在有限的步骤之后会自动结束而不会无限循环
    算法中的每一步都有明确的含义，不会出现二义性
    算法的每一步是可行的
算法效率的度量
    影响算法效率的主要因素：
        算法采用的策略和方法
        问题的输入规模
        编译器所产生的代码
        计算机执行速度
    事后统计法：比较不同算法对同一组输入数据的运行处理时间
        缺陷：实施困难，必须编写相应的程序，运行时间严重依赖硬件以及运行时的环境因素，算法的测试数据的选取相当困难
        优点：直观
    事前分析估算：依据统计的方法对算法效率进行估算
        理论依据：
            算法最终要编译成具体的计算机指令
            每条指令，在具体的计算机cpu上运行的时间是固定的
            通过运行指令数量的多少就可以推导出算法的复杂度
        典型代表：大O表示法
算法时间复杂度(大O表示法)
    n表示问题规模，常见的算法效率有：O(1),O(n),O(n平方)等
    算法效率严重依赖操作数量(指令的执行数量)
    操作数量可以作为时间复杂度的估算
    在判断时首先关注操作数量的最高次项
    判断一个算法的效率时，往往只需要关注操作数量的最高此项，其它次要项和常数项可以忽略，所以：O(n)和O(3n+1)等价
    在没有特殊说明时，我们所分析的算法时间复杂度是指最坏时间复杂度
    常见时间复杂度
    |----------------------------------|--------------------------|------------------------------|
    |           操作数量函数            |         阶               |          非正式术语           |
    |----------------------------------|--------------------------|------------------------------|
    |    12                            |      O(1)                |          常数阶              |
    |    2n+3                          |      O(n)                |          线性阶              |
    |    3n平方+2n+1                   |      O(n平方)             |          平方阶              |
    |    5log2n+20                     |      O(logn)             |          对数阶              |
    |    2n+3nlog2n+1                  |      O(nlogn)            |          nlogn阶             |
    |    6n三次方+2n平方+3n+4           |      O(n三次方)           |          立方阶              |
    |    2的n次方                      |      O(2的n次方)          |          指数阶              |
    |----------------------------------|--------------------------|------------------------------|
    O(1)时间复杂度最小，各个阶的复杂度比较：
        O(1)<O(logn)<O(n)<O(nlogn)<O(n平方)<O(n立方)<O(2的n次方)<O(n!)<O(n的n次方)
算法空间复杂度(大O表示法)
    n表示问题规模，常见的算法效率有：O(1),O(n),O(n平方)等
    对内存空间需求可以作为空间复杂度的估算
数据的运算
    在数据的逻辑结构上定义的操作，它在数据的存储结构上实现
    插入
    删除
    修改
    查找
    排序
```
## 线性表
```
线性表是0个或多个数据元素的集合
线性表中的数据元素之间是有顺序的
线性表中的数据元素个数是有限的
线性表中的数据元素的类型必须相同
数学定义：
    线性表是具有相同类型的n(n>=0)个数据元素的有限序列，类似于数列，a0,a1,a2,....an
    a0为线性表的第一个元素，只有一个后继
    an为线性表的最后一个元素，只有一个前驱
    除a0和an外的其他元素ai,即有前驱，又有后继
    线性表能够逐项访问和顺序存取
基本操作：
    创建线性表
    销毁线性表
    清空线性表
    将元素插入线性表
    将元素从线性表删除
    获取线性表中指定位置的元素
    获取线性表的长度

线性表在程序中表现为一种特殊的数据类型
线性表的操作在程序中的表现为一组函数
```
